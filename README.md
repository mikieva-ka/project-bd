# Проект: база данных "Театр"
## О проекте 
Этот проект представляет собой реляционную базу данных, разработанную для моделирования работы театра. Система отслеживает:

Театры и их залы
Спектакли и их расписание (сеансы)
Жанровую принадлежность спектаклей
Продажу билетов на конкретные места и ряды
Профили зрителей (пользователей)
Актёрский состав и распределение ролей

База данных может служить основой для реального сервиса бронирования билетов или использоваться в учебных целях для изучения проектирования реляционных моделей и нормализации данных.

## Сущности (таблицы)
Таблица
Описание

spectakls

## Связи между сущностями
Схема связей построена на принципах реляционной целостности и включает следующие зависимости:

Theatrs → zals
Один театр содержит один или несколько залов (zals.theatr_id → Theatrs.theatr_id)
zals → spectakls
Один зал может быть местом проведения одного или нескольких спектаклей (spectakls.zal_id → zals.zal_id)
spectakls ↔ zhanrs
Многие-ко-многим: спектакль может относиться к нескольким жанрам и наоборот (spectakls_zhanrs — связующая таблица)
spectakls → seanses
Один спектакль может проходить в разные дни и время (seanses.spectakl_id → spectakls.spectakl_id)
seanses → bilets
На каждый сеанс продаются билеты (bilets.seans_id → seanses.seans_id)
users → bilets
Один пользователь может купить несколько билетов (bilets.user_id → users.user_id)
spectakls → roles ← actors
Роли связывают актёров со спектаклями:
roles.spectakl_id → spectakls.spectakl_id
roles.actor_id → actors.actor_id
